  这三个想法质量都不错，但值得逐个想清楚，因为它们对应的是不同类型的"智能"。
  
  ---
  
  ## 逐个分析
  
  ### 1. 复习 Agent
  
  这个最有 Agent 价值。
  
  ```
  用户说："帮我复习上周学的内容"
  
  Agent 需要：
    → 查历史学习记录（学了什么）
    → 判断哪些知识点遗忘风险高（遗忘曲线）
    → 决定出什么题、出多少
    → 根据答题结果，动态调整下一题
    → 最终给出复习报告
  ```
  
  每一步都依赖上一步的结果做决策，**流程不固定**，这正是 Agent 适合的场景。这个功能和 flashcard 结合，是知萃·Distill 真正的差异化壁垒。
  
  ---
  
  ### 2. 多 LLM 并行生成再比较
  
  这个严格来说**不是 Agent，是应用**，但是个好功能。
  
  ```
  固定流程：
    同一个转录 → DeepSeek 生成 / Qwen 生成 / Claude 生成
                → 比较结果 → 展示给用户
  ```
  
  流程完全确定，不需要 LLM 做决策。用 n8n 的并行节点就能实现，和你现在的架构一样。
  
  价值在于**产品层面**，不在于技术复杂度。而且你之前记录过这个想法作为付费增值服务——方向是对的，用户愿意为"哪个 AI 更好"付费。
  
  ---
  
  ### 3. 比较结果
  
  这里有个关键问题：**谁来比较？**
  
  - **用户自己比较**：并排展示，用户判断 → 简单，但用户累
  - **再用一个 LLM 来评判**：`评判 LLM` 看三个结果，给出哪个更好、为什么 → 这才开始有 Agent 味道
  
  后者有个业界名字叫 **LLM-as-Judge**，是目前评估 AI 输出质量的主流方法。
  
  ---
  
  ## 优先级建议
  
  | 功能 | 技术类型 | 难度 | 商业价值 |
  |------|------|------|------|
  | 复习 Agent | 真 Agent | 高 | 高，核心差异化 |
  | 多 LLM 并行 | 应用 | 低 | 中，付费增值点 |
  | LLM-as-Judge 评判 | 应用+少量推理 | 中 | 中，增强多 LLM 功能 |
  
  **建议顺序**：先把 flashcard 基础版跑通 → 多 LLM 并行（技术简单，商业价值快速验证）→ 复习 Agent（需要用户数据积累才有意义）。
  
  复习 Agent 要做好，前提是有**用户学习历史数据**，现在知萃没有持久化存储，这是要先解决的基础设施问题。
  
  要现在规划存储层，还是继续推进 flashcard？
